Multiples of 3 and 5

Time Limit: 2 seconds

Project Euler 1: https://projecteuler.net/problem=1

If we list all the natural numbers below 10 that are multiples of 3 or 5, we get 3, 5, 6 and 9. The sum of these multiples is 23.

Given some number NN, what is the sum of all multiples of three and five that are strictly less than NN?

Input

Each input will consist of a single integer on a single line giving N(10≤N≤10,000)N(10≤N≤10,000).

Output

Output a single number on a single line, giving the sum of all multiples of 3 or 5 below NN.

Sample Input 1 Sample Output 1 
10

23
Sample Input 2 Sample Output 2 
1000

233168
Sample Input 3 Sample Output 3 
381

33758

Anagrams

Time Limit: 2 seconds

This problem will ask you to recognize if two strings are anagrams of each other. Two strings are anagrams of each other if you can rearrange the letters of one to make the other (They have exactly the same letters, order not mattering).

Input

Input will consist of a single test case. You will be provided with two strings, aa and bb. Both strings will consist of lowercase alphabetical characters only.

Output

Print “‘true”’ if aa and bb are anagrams of each other, “‘false”’ otherwise.

Sample Input 1 Sample Output 1 
tommarvoloriddle
iamlordvoldemort
 
true
Sample Input 2 Sample Output 2 
zigging
zagging
 
false

Removing Duplicates

Time Limit: 2 seconds

You will be provided a list of integer numbers, you must output these numbers in the order you recieve them, but making sure you remove any duplicates.

Input

Input will consist of one test case. Input will begin with a single integer NN, that will specify the number of integers that will be in the list. This will be followed by NN integers on a single line.

Output

Output the numbers in the order they appear in the input, but removing the duplicate entries.

Sample Input 1 Sample Output 1 
10
2 3 5 1 2 4 1 5 2 6
 
2 3 5 1 4 6

Paths through a Tree

Time Limit: 2 seconds

Consider a binary tree with each node having a string value. Each child has a unique path from the root down to the child. Print the path to every child in the tree by traversing from the root. Include both the root and the child in your path. Traverse left nodes first.

In the given illustration, the possible paths are (in order):

  • “top hat”

  • “top of the world”

  • “top of the morning”

Input

Each input will be a single binary tree, with no more than 1,0001,000 nodes. The description is recursive, and given as follows:

A leaf node will be given as a single word consisting of only lowercase (a-z) characters giving its value. An internal node will begin with the character [, followed by at least one space, followed by a word (as defined before) denoting the value of the internal node. Two tree descriptions, left then right, will follow, separated by at least one space. The character - may be given in place of a valid tree description for either left or right, denoting that there is no left or right subtree (a null value). Finally, a closing ] will be given. The children of an internal node may be leaf nodes or internal nodes themselves.

The tree will be on a single line.

Take note of the sample input.

Output

Output all the paths to the children. Separate the values of the nodes with a single space. Each path should be on its own line.

Order matters, print out paths of left children before the paths of right children. See the sample output.

Sample Input 1 Sample Output 1 
[ top hat [ of [ the world morning ] - ] ]

top hat
top of the world
top of the morning
Sample Input 2 Sample Output 2 
[ a [ b - [ c [ d [ e - q ] f ] b ] ] [ h [ i [ j a - ] - ] z ] ]

a b c d e q
a b c d f
a b c b
a h i j a
a h z

Subsequence Sums

Time Limit: 2 seconds

Given an array and a target value, find out if there is a continuous subsequence that sums to the target value. Eg. Given: [1 2 3 4 5], 8, the answer would be false, with a target value of 10 it would be true using (1 2 3 4), a target value of 9 could be true either using (2 3 4) or (4 5), and a target of 7 would be true using (3 4). A target value of 1 would also be true, as you can just use (1).

Input

Input will consist of a single test case. The first line of input will contain two integers, NN and TT, specifying the number of elements in the array and the target value. This will be followed by NN positive integers.

Output

Output “‘true”’ if there is a subsequence that sums to the target “‘false”’ otherwise.

Sample Input 1 Sample Output 1 
5 10
1 2 3 4 5
 
true
Sample Input 2 Sample Output 2 
5 8
1 2 3 4 5
 
false
Sample Input 3 Sample Output 3 
5 8
4 3 1 2 5
 
true

Friend Groups

Time Limit: 4 seconds

Let’s define a friend group as a number of people who can be connected by mutual friends. For example, take four people: Alice, Bob, Carter, and David. Say that Alice and Bob are friends; Carter and David are friends; and Bob and Carter are friends. Then we can say that Alice, Bob, Carter, and David form a friend group of four people. However, if Bob and Carter were not friends, then there would be two distinct friend groups: Alice and Bob would be separate from Carter and David.

Given a list of people and their friendships, can you determine the number of distinct friend groups?

A person with no friends is considered to be in their own friend group.

Input

Input will begin with a line containing two integers N(1≤N≤5000)N(1≤N≤5000), and F(1≤F≤50000)F(1≤F≤50000) denoting the number of people and the number of “friendship” pairs respectively. The following line will contain NN names (consisting of only upper-case and lower-case characters) each separated by a single space. FF lines will follow, all of the form aa bb, with aa, bb being names specified in the second line, denoting that aa and bb are friends.

Output

Output a single number on a single line denoting the number of distinct friend groups.

Sample Input 1 Sample Output 1 
4 3
Alice Bob Carter David
Alice Bob
Carter David
Bob Carter

1
Sample Input 2 Sample Output 2 
4 2
Alice Bob Carter David
Alice Bob
Carter David

2
